Introduction

This library was created to allow developers to save time by taking the complexity out of building forms. With FormulationJS, anyone can quickly build web forms using JSON objects and render them in ReactJS. The generated forms can be integrated with third-party applications for both desktops and mobile browsers. Keep an eye out for a more comprehensive set of inputs and other functional form features, including a UI to design forms without JSON and API's to get, update, or save submitted data.

Setting Up

Download the Repository

Option 1 - Cloning Git Repository Using HTTPs

  1. git clone git@github.com:codazen/formulationjs.git

Option 2 - Download the ZIP

  1. Visit our GitHub Repository and click the "Download ZIP" button.
  2. Unzip the folder.
Installation

To install and run a demo version of the project...

  1. Install node if you don't already have it - https://docs.npmjs.com/getting-started/installing-node
  2. In your terminal, go to the downloaded formulationjs directory and run the following commands: {% highlight bash %} npm install npm run build-sample1 {% endhighlight %}
  3. Open a new terminal window in the same directory and run the following command: {% highlight bash %} npm start {% endhighlight %}
  4. The project has been installed and is now running.
  5. To view the demo, go to http://localhost:3000/sample1/ in your browser.

Form Definition

Build a new form using the following properties.
Every form can hold their own set of elements, allowing users to input values to submit through the Form.
Please note that all field names defined are case sensitive.


Form Properties

Field Name Field Type Description
name string This value is displayed as the name or title of the form.
body string This value is displayed as the form's description text.
submitDisabled boolean Allows users to disable the submit button. By default, the submit button is enabled.
elements object array This property will hold a list of form elements that will be rendered as input fields in the form. See definitions of the form elements below.

Element Properties

Field Name Field Type Description
type string The type of form element we want to render. Currently supports 'textbox', 'textarea', 'checkbox', 'dropdown', and 'datepicker'.
data Object This object holds a list of parameters to render a form element. The available parameters will vary depending on the element type. See the different form element object definitions below.

Rendered Form

Textbox

Render a single-line textbox field.


Properties

Textbox Data Object
Field Name Field Type Description
label string This value is displayed as the label for the textbox.
value string This value is pre-populated in the text input when the form is loaded.
required boolean Determines whether or not a user must fill out this component in order to submit the form.
maxlength string This value represents the maximum character limit for the component.
textboxState boolean If false, disables the input textbox.
email boolean Adds email format validation to the input textbox.

Code

{% highlight json %}
{
  "type": "textbox",
  "data": {
    "value": "",
    "label": "Label",
    "required": true,
    "maxlength": "10",
    "email": true,
    "textboxState": true
  }
}
{% endhighlight %}

Rendered Element

Checkbox List

Render a checkbox list field.


Properties

Checkbox List Data Object
Field Name Field Type Description
label string This value is displayed as the label for the checkbox list.
value array An array that contains the initial check box values that are checked.
required boolean Determines whether or not a user must check at least one of the items in the checkbox list in order to submit the form.
options object array An array to define the list of checkbox options.
Checkbox List Options Object
Field Name Field Type Description
name string Text that will be displayed as the checkbox option.
value string | int The actual value submitted if the option is checked.

Code

{% highlight json %}          
{
  "type": "checkbox",
  "data": {
    "value": ["1"],
    "groupLabel": "Checkbox group label",
    "required": true,
    "options": [
      {
        "name": "Option 1",
        "value": 1
      },
      {
        "name": "Option 2",
        "value": 2
      }
    ]
  }
}
{% endhighlight %}

Rendered Element

Dropdown List

Render a dropdown list field.


Properties

Dropdown List Data Object
Field Name Field Type Description
label string This value is displayed as the label for the dropdown list.
value string Should match the "value" of a dropdown option to indicate the default selected option. If no value is specified here, the placeholder will be selected by default.
required boolean Determines whether or not a dropdown selection must be made in order to submit the form.
placeholder string This value is displayed as the first default selection of the drop down list. The placeholder is not considered a valid selection for the required field validation.
options object array An array to define the list of dropdown options.
Dropdown List Options Object
Field Name Field Type Description
name string Text that will be displayed as a dropdown option.
value string | int The actual value submitted if the option is selected.

Code

{% highlight json %}
{
  "type": "dropdown",
  "placeholder": "Select...",
  "data": {
    "label": "Dropdown Label",
    "options": [
      {
        "name": "Option 1",
        "value": 1
      },
      {
        "name": "Option 2",
        "value": 2
      }
    ],
    "value": "",
    "required": true
  }
}
{% endhighlight %}

Datepicker

Render a datepicker that allows the user to choose a specific date.
This datepicker utilizes the ReactJS Datepicker library - http://hacker0x01.github.io/react-datepicker/.


Properties

Datepicker Data Object
label string A label rendered adjacent to the checkbox input.
value boolean The initial value of the datepicker input.
required boolean A user must fill out this component in order to submit the form.
maxlength string This value represents the maximum character limit for the component.
range boolean This functionality is not yet available.
placeholderText string This value is displayed as placeholder text in the text area.
dateFormat string
  • Allows user to setup specific date format.
  • Separator symbols are customize-able and optional.
  • Days, Months, and Years can be used with any combination of the valid dateFormat symbols (shown below).
Valid dateFormat Symbols
YY Last 2 digits of year.
YYYY 4 digit year.
M Number representative of month.
MM Two digit number representative of month.
MMM First three letters of spelled out name of month.
MMMM Spelled out name of month.
D Number representative of day of month.
DD Two digit number representative of day of month.
dd First two letters of the spelled out name of day.
ddd First three letters of the spelled out name of day.
dddd Spelled out name of day.

Code

{% highlight json %}
{
  "type": "datepicker",
  "data": {
    "label": "Please select a due date",
    "value": null,
    "dateFormat": "DD/MM/YYYY",
    "placeholderText": "dd/mm/yyyy",
    "maxlength": "10",
    "required": "false",
    "range": "false"
  }
}
{% endhighlight %}
